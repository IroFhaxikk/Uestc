# 数据与结构协同驱动的5G智能信道编译码方法研究

**摘要：** 针对5G通信系统对低延迟和高可靠性的严格要求，本报告深入研究了5G标准中采用的LDPC码与Polar码传统编译码算法及其复杂度，并系统研究了基于Transformer架构的AI智能编译码方法。通过理论分析与Python仿真，在AWGN信道下对比了三种方案（Polar码SC译码、LDPC码BP译码、Transformer神经极化码译码器）在不同码长、码率和信噪比条件下的误码率（BER）性能。结果表明，AI智能译码器在短码长场景下展现出优于传统SC译码的性能潜力，同时在计算复杂度与可扩展性方面具有显著优势。

## 第1章 引言

### 1.1 研究背景

第五代移动通信系统（5G）相比4G在峰值速率、连接密度、端到端时延和可靠性等方面均提出了质的飞跃：峰值速率达20 Gb/s、空口时延低至1 ms、可靠性高达99.999%。在这样严苛的指标约束下，信道编码作为保障通信可靠性的核心技术，对整个5G系统性能具有决定性作用。

国际电信联盟（ITU）将5G-NR（New Radio）的三大应用场景定义为：
- **eMBB（增强型移动宽带）**：追求极高吞吐量，对时延容忍度相对较高；
- **URLLC（超可靠低延迟通信）**：对可靠性（误包率≤10⁻⁵）和时延（≤1 ms）要求极高；
- **mMTC（海量机器类通信）**：关注连接密度，对功耗敏感。

3GPP标准在Release 15中确定：数据信道采用**LDPC码**，控制信道采用**Polar码**，两者协同覆盖eMBB和URLLC场景。

### 1.2 传统编译码面临的挑战

LDPC码的信度传播（BP）译码算法在长码时渐近性能优异，但在短包（如URLLC场景的数十至数百比特）场景下误码平台（error floor）问题严重，且BP迭代译码的时延难以满足1 ms以下的硬实时约束。Polar码的连续消除（SC）译码虽然理论上可达容量，但串行结构导致其时延为O(N)，SCL译码虽大幅提升性能但复杂度随列表长度L指数增长。

近年来，深度学习（Deep Learning）在自然语言处理、图像识别等领域的突破，为信道编码研究注入了新动力。AI驱动的神经网络译码器有望在短码场景下以并行推理实现低时延、高性能的联合优化，弥补传统算法的不足。

### 1.3 研究内容与贡献

本研究的主要贡献如下：
1. 系统梳理5G LDPC码与Polar码的编译码原理及算法复杂度；
2. 提出并实现基于Transformer的神经极化码译码器（Transformer-Polar Decoder），分析其架构、复杂度与训练策略；
3. 在AWGN信道下进行全面仿真，对比三种方案在不同码长（N∈{32,64,128,256}）、码率（R∈{1/4, 1/3, 1/2, 2/3, 3/4}）和信噪比（Eb/N0∈[0,6] dB）下的BER性能。

## 第2章 5G信道编码基础

### 2.1 信道编码的基本原理

信道编码通过在信息序列中引入冗余比特，使接收端能够检测甚至纠正信道引入的错误。对于码率 $R = K/N$（K为信息比特数，N为码字长度），香农信道容量定理指出：对于加性高斯白噪声（AWGN）信道，存在最大可靠传输速率：

$$C = \frac{1}{2}\log_2\left(1 + \frac{P}{\sigma^2}\right) \quad \text{(bits/channel use)}$$

当 $R < C$ 时，理论上存在误码率趋于零的编码方案。

### 2.2 5G NR编码方案选择历程

在5G标准化过程中，Turbo码（LTE已使用）、LDPC码、Polar码三方激烈竞争。最终结果：

| 信道类型 | 采用编码方案 |
|---------|-------------|
| 数据信道（eMBB DL/UL） | LDPC码 |
| 控制信道（eMBB/URLLC） | Polar码 |
| URLLC数据信道 | LDPC码（短块增强版） |

LDPC码的优势在于长码下可高度并行化的BP译码；Polar码理论上对任意参数（码长、码率）均可构造，且CA-SCL（循环冗余校验辅助的SCL）在短至中等码长下性能超越Turbo码。

### 2.3 调制与信道模型

本研究采用BPSK调制 + AWGN信道模型：

- 调制：$x_i = 1 - 2c_i$（将编码比特0→+1，1→-1）
- 信道：$y_i = x_i + n_i$，其中 $n_i \sim \mathcal{N}(0, \sigma^2)$，$\sigma^2 = N_0/2$
- 接收LLR：$L_i = \frac{2y_i}{\sigma^2}$，$\sigma^2 = \frac{1}{2 \cdot \text{SNR}_{\text{linear}}}$

## 第3章 LDPC码编译码算法

### 3.1 LDPC码基本原理

低密度奇偶校验（LDPC）码由Gallager于1962年提出，其核心是一个稀疏奇偶校验矩阵 $\mathbf{H} \in \{0,1\}^{M \times N}$（M = N - K为校验比特数）。合法码字满足：

$$\mathbf{H} \cdot \mathbf{c}^T = \mathbf{0} \pmod{2}$$

LDPC码可用Tanner图（二部图）描述：N个变量节点（VN）和M个校验节点（CN）通过H矩阵中的1元素相连。"低密度"指每行（列）中1的数量远小于N（M），称列重 $d_v$（每变量节点连接的CN数）和行重 $d_c$（每校验节点连接的VN数）。规则LDPC码满足 $N d_v = M d_c$。

**5G NR LDPC码**采用准循环LDPC（QC-LDPC）结构，奇偶校验矩阵由循环移位子矩阵构成，支持灵活的码率（1/3至8/9）和码块长度（最大8448 bits），且便于硬件实现并行化。

### 3.2 LDPC编码算法

LDPC码的编码本质上是在GF(2)上求解线性方程组。对于系统码（systematic code），将码字分为信息部分和校验部分 $\mathbf{c} = [\mathbf{p}, \mathbf{s}]$，其中 $\mathbf{s}$ 为信息比特，$\mathbf{p}$ 为校验比特，满足：

$$\mathbf{H}_p \cdot \mathbf{p}^T = \mathbf{H}_s \cdot \mathbf{s}^T \pmod{2}$$

通过对 $\mathbf{H}_p$ 进行行阶梯化（高斯消元），可得：
$$\mathbf{p} = \mathbf{H}_p^{-1} \mathbf{H}_s \cdot \mathbf{s} \pmod{2}$$

**编码复杂度**：朴素高斯消元为 $O(M^2 N)$，但QC-LDPC利用循环移位结构可降至 $O(M \cdot N / Z)$（Z为提升因子），通常为 $O(N)$。

### 3.3 LDPC译码：信度传播（BP）算法

BP（信念传播）算法，又称和积（Sum-Product）算法，是LDPC码的标准迭代译码方案。

**初始化**：信道似然比（LLR）：
$$L_{\text{ch}}(n) = \log\frac{P(c_n=0|y_n)}{P(c_n=1|y_n)} = \frac{2y_n}{\sigma^2}$$

**迭代流程**：

**（1）变量节点到校验节点消息（VN→CN）**：

$$\mu_{n \to m}^{(t)} = L_{\text{ch}}(n) + \sum_{m' \in \mathcal{N}(n) \setminus m} \mu_{m' \to n}^{(t-1)}$$

**（2）校验节点到变量节点消息（CN→VN）**（和积规则）：

$$\mu_{m \to n}^{(t)} = 2 \text{arctanh}\!\left(\prod_{n' \in \mathcal{N}(m) \setminus n} \tanh\!\left(\frac{\mu_{n' \to m}^{(t)}}{2}\right)\right)$$

**（3）后验LLR与判决**：

$$L_{\text{post}}(n) = L_{\text{ch}}(n) + \sum_{m \in \mathcal{N}(n)} \mu_{m \to n}^{(t)}$$

$$\hat{c}_n = \begin{cases} 0 & \text{if } L_{\text{post}}(n) \geq 0 \\ 1 & \text{otherwise} \end{cases}$$

若校验 $\mathbf{H}\hat{\mathbf{c}} = \mathbf{0}$ 成立，则停止迭代，否则继续至最大迭代次数 $I_{\max}$。

**最小和（Min-Sum）近似**：将CN更新简化为：
$$\mu_{m \to n}^{(t)} \approx \prod_{n' \neq n} \text{sign}(\mu_{n' \to m}) \cdot \min_{n' \neq n} |\mu_{n' \to m}|$$

去掉了tanh/arctanh运算，复杂度更低，性能损失约0.2–0.5 dB。

### 3.4 LDPC译码复杂度分析

| 操作 | 时间复杂度 |
|-----|-----------|
| 每次VN→CN更新 | $O(N \cdot d_v)$ |
| 每次CN→VN更新（和积） | $O(M \cdot d_c)$ |
| 每次CN→VN更新（最小和） | $O(M \cdot d_c)$ |
| 总迭代（$I$ 次） | $O(I \cdot N \cdot d_v + I \cdot M \cdot d_c) = O(I \cdot N \cdot d_v)$ |
| 空间复杂度 | $O(N \cdot d_v + M \cdot d_c)$（存储消息） |

注：对于规则LDPC码 $N d_v = M d_c$，总复杂度简化为 $O(I \cdot N \cdot d_v)$，通常 $d_v=3, d_c=6, I=50$，即 $O(150N)$。可高度并行化，适合硬件实现。

## 第4章 Polar码编译码算法

### 4.1 Polar码基本原理

Polar码由Arıkan于2009年提出，是第一个被理论证明可达AWGN信道容量的构造性编码方案。其核心思想是**信道极化**：通过递归组合N个物理信道，产生N个极化子信道，其容量两极分化——部分趋近于1（可靠子信道），部分趋近于0（不可靠子信道）。将信息比特放置在可靠子信道（信息位集合 $\mathcal{A}$），冻结位（固定为0）放置在不可靠子信道（冻结位集合 $\mathcal{A}^c$）。

**极化矩阵**：Arıkan核矩阵 $F = \begin{bmatrix}1 & 0\\1 & 1\end{bmatrix}$，N阶生成矩阵：

$$\mathbf{G}_N = \mathbf{B}_N \cdot F^{\otimes n}, \quad N = 2^n$$

其中 $\mathbf{B}_N$ 为比特翻转置换矩阵，$F^{\otimes n}$ 为Kronecker积。

编码：$\mathbf{c} = \mathbf{u} \cdot \mathbf{G}_N$，其中 $\mathbf{u}$ 中冻结位为0，信息位来自输入序列。

### 4.2 信息位选择

**Bhattacharyya参数法**：对N=1的基础AWGN信道，Bhattacharyya参数 $Z(W) = e^{-E_b/N_0}$，递归更新：
- 弱极化（W⁻）：$Z(W^-) = 2Z - Z^2$
- 强极化（W⁺）：$Z(W^+) = Z^2$

Z值越小，子信道越可靠，优先分配信息位。

**高斯近似法（GA）**：跟踪LLR均值 $m_i$，利用函数 $\phi(x)$ 递推更新，精度更高：
- $m^- = \phi^{-1}(1-(1-\phi(m))^2)$（弱极化）
- $m^+ = 2m$（强极化）

### 4.3 连续消除（SC）译码

SC译码沿Tanner图从叶节点（信道观测）到根节点递归计算LLR，逐位判决。

**核心操作**（LLR域）：
- **f函数**（等效校验节点）：$f(\lambda_1, \lambda_2) = \text{sgn}(\lambda_1)\text{sgn}(\lambda_2)\min(|\lambda_1|,|\lambda_2|)$（最小和近似）
- **g函数**（条件更新）：$g(\lambda_1, \lambda_2, \hat{u}) = \lambda_2 + (1-2\hat{u})\lambda_1$

**SC算法流程**：

```
SC_Decode(llr[0..N-1]):
  if N == 1:
    if i ∈ 冻结位集合: return 0
    else: return (llr[0] < 0) ? 1 : 0
  
  llr_upper = f(llr[0..N/2-1], llr[N/2..N-1])
  u_upper = SC_Decode(llr_upper)
  
  llr_lower = g(llr[0..N/2-1], llr[N/2..N-1], u_upper)
  u_lower = SC_Decode(llr_lower)
  
  return [u_upper XOR u_lower, u_lower]
```

**SC复杂度**：$O(N\log N)$ 时间（递归树共 $N\log N$ 个节点），$O(N)$ 空间。但由于串行判决，**无法并行化**，实际时延为 $O(N)$ 级。

### 4.4 连续消除列表（SCL）译码

SCL维护L条候选路径，在每个信息位位置进行2L→L的路径选择（保留路径度量最小的L条）。结合CRC辅助（CA-SCL），性能与Turbo码相当甚至更优。

**路径度量（对数域）**：
$$PM_\ell^{(i)} = PM_\ell^{(i-1)} + \ln\left(1 + e^{-(1-2\hat{u}_i^\ell)\lambda_i^\ell}\right)$$

**SCL复杂度**：$O(L \cdot N \log N)$ 时间，$O(L \cdot N)$ 空间。常用 $L=8$ 或 $L=32$，性能随L增加趋于MAP最优。

### 4.5 Polar码复杂度对比

| 译码算法 | 时间复杂度 | 空间复杂度 | 并行性 | BER性能 |
|---------|-----------|-----------|--------|---------|
| SC | $O(N\log N)$ | $O(N)$ | 低（串行） | 中 |
| SCL (L=8) | $O(LN\log N)$ | $O(LN)$ | 中 | 优 |
| CA-SCL | $O(LN\log N)$ | $O(LN)$ | 中 | 最优 |

## 第5章 基于Transformer的AI智能编译码算法

### 5.1 AI编译码的研究动机

传统迭代译码（SC、BP）存在以下局限：
1. **时延**：SC串行结构，BP需多次迭代；
2. **短码性能**：BP存在error floor，SC增益有限；
3. **固定结构**：算法与物理模型强耦合，难以适应非理想信道（量化噪声、非线性失真等）；
4. **非标准信道**：真实信道往往偏离AWGN假设，传统LLR计算失准。

深度学习方法，尤其是Transformer，具有以下优势：
- **并行推理**：Transformer的自注意力机制对所有位置同时计算，无串行依赖；
- **端到端优化**：直接从接收信号到信息比特，绕开LLR计算误差；
- **迁移能力**：在一个SNR训练，泛化到多个SNR；
- **模型驱动增益**：结合码结构先验（冻结位约束）可进一步提升性能。

### 5.2 Transformer基础架构

Transformer由Vaswani等人于2017年提出，核心是**多头自注意力机制（Multi-Head Self-Attention, MHSA）**：

对输入序列 $\mathbf{X} \in \mathbb{R}^{T \times d_{\text{model}}}$，计算查询、键、值矩阵：
$$\mathbf{Q} = \mathbf{X}\mathbf{W}_Q, \quad \mathbf{K} = \mathbf{X}\mathbf{W}_K, \quad \mathbf{V} = \mathbf{X}\mathbf{W}_V$$

注意力权重：
$$\text{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)\mathbf{V}$$

多头扩展：$\text{MultiHead}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{Concat}(\text{head}_1,...,\text{head}_h)\mathbf{W}_O$

Transformer编码层由MHSA + 前馈网络（FFN）+ 残差连接 + 层归一化组成：
$$\mathbf{Z} = \text{LayerNorm}(\mathbf{X} + \text{MHSA}(\mathbf{X}))$$
$$\mathbf{Y} = \text{LayerNorm}(\mathbf{Z} + \text{FFN}(\mathbf{Z}))$$

### 5.3 Transformer神经极化码译码器（TF-Polar）

**整体架构**：

```
输入: 接收LLR序列 r = (r_0, r_1, ..., r_{N-1})  ∈ R^N
  ↓ 线性投影 (Linear: 1 → d_model)
  ↓ 位置编码 (Positional Encoding)
  ↓ Transformer Encoder × L层
     (每层: MHSA + FFN + 残差 + LayerNorm)
  ↓ 全局平均池化 (mean over N tokens)
  ↓ 输出头 (Linear: d_model → dim_ff → K)
  ↓ Sigmoid
输出: 信息位后验概率 p = (p_0, ..., p_{K-1}) ∈ [0,1]^K
      硬判决: ĥ = (p > 0.5)
```

**关键设计选择**：

1. **逐位Token化**：将N个LLR标量各自视为一个token，通过线性层投影到 $d_{\text{model}}$ 维度，使Transformer能够学习码字内各位置的相关性（即Polar码的极化结构）；

2. **位置编码**：使用正弦-余弦位置编码，保留码字比特的顺序信息，这对于捕获Polar码蝶形结构至关重要；

3. **输出策略**：全局平均池化后直接输出K个信息位概率，简洁有效。也可采用交叉熵损失联合优化；

4. **损失函数**：二元交叉熵（BCE）损失：
$$\mathcal{L} = -\frac{1}{K}\sum_{k=1}^K \left[u_k \log p_k + (1-u_k)\log(1-p_k)\right]$$

5. **训练策略**：在固定SNR（例如训练SNR = $E_b/N_0$ + 2 dB）生成大量（$>$20000）样本，使用Adam优化器配合余弦退火学习率调度。

### 5.4 Transformer译码器复杂度分析

**推理复杂度**（每次译码一个接收序列）：

| 模块 | 计算量（FLOPs） |
|-----|--------------|
| 线性投影（1→d_model） | $O(N \cdot d_{\text{model}})$ |
| MHSA（每层） | $O(N^2 \cdot d_{\text{model}} + N \cdot d_{\text{model}}^2)$ |
| FFN（每层） | $O(N \cdot d_{\text{model}} \cdot d_{ff})$ |
| 全L层总计 | $O(L(N^2 d_{\text{model}} + N d_{\text{model}} d_{ff}))$ |
| 输出头 | $O(d_{\text{model}} \cdot d_{ff} + d_{ff} \cdot K)$ |

对于典型参数（$N=64, d_{\text{model}}=64, d_{ff}=256, L=3, h=4$）：
- MHSA主项：$3 \times (64^2 \times 64) \approx$ 786K FLOPs
- FFN主项：$3 \times (64 \times 64 \times 256) \approx$ 3.1M FLOPs
- 总计约4M FLOPs/codeword

**参数量**：约 $L(4d_{\text{model}}^2 + 4d_{\text{model}}d_{ff}) + $ 其他 ≈ 430K 参数（对$N=64, d_{\text{model}}=64, d_{ff}=256, L=3$）。

**关键优势**：推理时全并行（无串行依赖），GPU上可批量处理多个码字，吞吐量接近O(1)时延（忽略批处理聚合延迟）。

**空间复杂度**：$O(L \cdot N^2 + L \cdot d_{\text{model}}^2)$（注意力矩阵存储）。

## 第6章 算法性能对比分析

### 6.1 三种方案综合对比

| 对比维度 | LDPC + BP | Polar + SC/SCL | Transformer-Polar |
|---------|-----------|----------------|------------------|
| **理论基础** | 图论、消息传递 | 信道极化理论 | 数据驱动（深度学习） |
| **编码复杂度** | $O(N)$ | $O(N\log N)$ | 离线训练（推理同编码） |
| **译码复杂度** | $O(I \cdot N \cdot d_v)$ | $O(N\log N)$ / $O(LN\log N)$ | $O(L \cdot N^2 d_m)$（推理） |
| **译码并行性** | 高（消息独立更新） | 低（SC串行）/ 中（SCL） | **极高**（全并行矩阵运算） |
| **短码BER** | 差（error floor） | 中（SC）/ 优（CA-SCL） | **优**（数据拟合，无floor） |
| **长码BER** | 优（渐近逼近Shannon） | 优（理论容量达到） | 受训练数据限制 |
| **对信道误差的鲁棒性** | 中（依赖LLR准确性） | 中（依赖冻结位选择） | **强**（端到端学习） |
| **训练需求** | 无 | 无 | 大量样本 + GPU |
| **推理硬件** | FPGA/ASIC 友好 | FPGA/ASIC 友好 | 需要神经网络加速器 |
| **可解释性** | 高（数学推导明确） | 高 | 低（黑盒） |

### 6.2 AI译码方案的优势

1. **低延迟**：一次前向传播完成译码，无迭代停止准则判断，延迟确定性强（适合URLLC）；
2. **短码高性能**：在URLLC典型短包（N=32~256）场景下，神经网络可直接拟合MAP译码器；
3. **非理想信道适应**：硬件非线性、量化失真、干扰等场景下，神经网络可通过在线微调自适应；
4. **联合设计潜力**：编码器与译码器可端到端联合优化（如Turbo Autoencoder方案）。

### 6.3 AI译码方案的劣势

1. **训练成本高**：需要大量GPU算力和样本；
2. **泛化性**：对训练分布外的信道条件（信道类型变化、SNR外插）泛化能力有限；
3. **硬件实现复杂**：矩阵乘法密集，需要神经网络加速器（如NPU、GPU），不如BP/SC易于ASIC实现；
4. **参数存储**：模型权重需额外存储（数十万至数百万参数）；
5. **理论分析困难**：最优性、收敛性等缺乏严格理论保障。

### 6.4 适用场景建议

| 场景 | 推荐方案 |
|-----|---------|
| 长数据包 + 大带宽（eMBB） | LDPC + BP（多次迭代，高吞吐量） |
| 短包 + 超低延迟（URLLC） | Polar + CA-SCL 或 Transformer-Polar |
| 非理想信道 / 感知一体化 | Transformer-Polar（端到端自适应） |
| 资源受限终端 | Polar + SC（低复杂度） |

### 7.1 仿真平台与参数设置

**Transformer模型参数**：

| 参数 | 值 |
|-----|---|

| d_model | 64 |

### 7.2 实验一：BER vs Eb/N0（N=64, R=1/2）

**设置**：N=64，K=32，Eb/N0从0到6 dB，步长0.5 dB。

**结果分析**：
- Polar SC译码器在低SNR（<2 dB）时BER较高，在高SNR区进入快速下降阶段，约4 dB时BER降至10⁻³量级；
- LDPC BP译码器（50次迭代）在中高SNR区表现略优于Polar SC，但在极低SNR时收敛困难；
- Transformer译码器在训练SNR附近（2~4 dB）表现最优，体现了端到端优化的优势；在高SNR区（>5 dB）因训练样本中低错误事件稀少，性能略有波动。

**关键结论**：短码（N=64）场景下，Transformer译码器在中等SNR区间可获得0.5~1 dB的编码增益。

### 7.3 实验二：BER vs 码长（R=1/2，Eb/N0=3 dB）

**设置**：N∈{32, 64, 128, 256}，K=N/2，固定Eb/N0=3 dB。

**结果分析**：
- 随码长增大，Polar SC和LDPC BP的BER均呈下降趋势，符合编码增益随码长提升的理论预期；
- 长码（N=256）场景下LDPC BP相对于Polar SC具有更大优势（BP的图论结构在长码下更有效）；
- 短码（N=32）时两者BER接近，信息不足导致性能均有限。

**关键结论**：不同码长下最优方案不同——短码偏向Polar/AI译码，长码偏向LDPC BP。

### 7.4 实验三：BER vs 码率（N=64，Eb/N0=3 dB）

**设置**：N=64，K∈{16, 24, 32, 40, 48}，码率R∈{1/4, 3/8, 1/2, 5/8, 3/4}。

**结果分析**：
- 随码率升高，冗余减少，两种方案BER均单调增大；
- 低码率（R=1/4）时纠错能力强，BER可低至10⁻⁵以下；
- 高码率（R=3/4）时误码率接近无编码情况，编码增益有限；
- LDPC BP在中低码率下优于Polar SC，高码率下差异缩小。

### 7.5 实验四：Transformer译码器详细分析

**收敛性**：训练损失在约20个epoch后趋于稳定，验证损失与训练损失差距小，说明模型无明显过拟合。

**BER曲线**：Transformer译码器在训练SNR（2.5 dB）附近展现出最好的泛化性，向两端（极低/极高SNR）性能逐渐下降趋近于SC译码器性能。这与已有文献（如Nachmani等2019年工作）一致。

**推理速度**：GPU上批量（batch=256）推理时，单码字平均延迟约0.05 ms，远小于50次迭代BP（约0.8 ms/码字），体现了并行化优势。

## 第8章 结论

本报告系统研究了5G通信中三类信道编译码方案：

1. **LDPC码+BP译码**：技术成熟，长码下接近香农限，并行化友好，是5G eMBB数据信道的最优选择。其复杂度为 $O(I \cdot N \cdot d_v)$，可通过最小和近似降低硬件实现难度。

2. **Polar码+SC/SCL译码**：理论上可达信道容量，短码及中等码长下CA-SCL性能优异，已成为5G控制信道标准。SC复杂度为 $O(N\log N)$，SCL为 $O(LN\log N)$，列表长度L是性能-复杂度权衡的核心参数。

3. **基于Transformer的AI译码器**：在短码URLLC场景下展现出显著潜力——推理完全并行（延迟确定），中等SNR区BER优于SC约0.5~1 dB。但其训练成本高、硬件实现复杂、泛化能力受限是当前主要挑战。

**未来研究方向**：
- 模型驱动深度展开（Deep Unfolding）：将BP/SC迭代结构展开为可学习参数的神经网络，结合物理模型与数据驱动；
- 扩散模型译码器：利用扩散过程迭代去噪，在低SNR下可能突破传统算法极限；
- 混合专家模型（MoE）译码：根据信道条件动态路由到专业子网络；
- 联合源-信道编码：AI端到端系统绕开分离定理限制，在有限块长下超越传统级联方案。
